# 统计分析与开票审核运维说明

本文档用于说明 `safety-bms` 当前已经上线的统计分析模块、权限接入方式，以及开票审核对仪表盘和统计口径的影响。

## 1. 统计分析模块

### 访问入口

- 前端页面：`/analytics`
- 后端路由前缀：`/api/v1/analytics`
- 访问权限：`analytics:read`

### 页面能力

统计分析页当前提供以下能力：

- **概览 KPI**：签约额、开票额、收款额、回款率、应收余额、逾期合同数
- **经营分析**：签约/开票/收款趋势、签约排行
- **财务分析**：应收账龄分布、高风险合同
- **客户分析**：客户增长趋势、行业分布、客户状态分布
- **服务分析**：服务效率趋势、服务类型分布
- **明细下钻**：点击图表或入口查看当前维度明细
- **导出**：按当前明细筛选结果导出 Excel
- **筛选**：支持按日期范围筛选，部分接口支持按服务类型筛选

### 指标口径

- **签约额**：统一按合同 `sign_date` 统计
- **开票额**：仅统计状态为 `issued`、`sent` 的发票
- **开票日期维度**：优先使用 `invoice_date`，为空时回退到 `created_at`
- **收款额**：按收款记录日期统计
- **权限口径**：统计接口、菜单和前端路由统一按入口权限控制，避免页面可见但接口无权访问或反之

### 中文标签

仪表盘与统计分析图表中的客户状态、合同状态、发票状态、服务类型等枚举值，前后端已统一按中文标签映射展示。若新增枚举，请同步更新后端枚举格式化逻辑与前端标签常量。

## 2. 权限与初始化

### 默认角色权限

`analytics:read` 默认授予以下角色：

- `admin`
- `sales`
- `service`
- `finance`
- `viewer`

### 新环境初始化

首次启动或本地开发环境初始化时，建议依次执行：

```bash
cd backend
PYTHONPATH=. python app/db/init_db.py
PYTHONPATH=. python app/cli/sync_permissions.py
```

说明：

- `init_db.py`：初始化默认角色、默认角色权限和管理员账号
- `sync_permissions.py`：将代码中的权限定义同步到数据库，并保证 `admin` 角色拥有全部权限

### 升级已部署环境

如果升级前环境中没有 `analytics:read` 权限记录，升级代码后需要执行：

```bash
cd backend
PYTHONPATH=. python app/cli/sync_permissions.py
```

否则前端菜单和路由虽然已经接入统计分析，但数据库里缺少权限记录时，角色授权和实际访问会不一致。

## 3. 开票审核流程

### 状态流转

- `applying`：申请中
- `issued`：审核通过并已开票
- `sent`：发票已寄出
- `rejected`：审核驳回

### 审核规则

- 审核通过时必须填写：
  - `invoice_date`
  - `actual_invoice_no`
- 审核驳回时必须填写：
  - `remark`（驳回原因）

### 金额校验

- 创建开票申请时会校验合同剩余可开票余额
- 审核通过时会再次校验，避免累计开票金额超过合同总额

## 4. 仪表盘与统计分析的开票口径

开票审核流上线后，仪表盘与统计分析中的“开票额”不再统计所有发票记录，而是仅统计真正进入业务结果态的发票：

- 计入：`issued`、`sent`
- 不计入：`applying`、`rejected`

这意味着开票申请创建后不会立刻反映到开票额图表，只有审核通过后才会进入统计结果。

  - `backend/app/cli/sync_permissions.py`
  - `frontend/src/config/menuConfig.ts`
